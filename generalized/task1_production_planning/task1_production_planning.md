# Задача 1. Производственное планирование

## Тип задачи
Линейное программирование (ЛП). Требуется определить объёмы выпуска нескольких
видов продукции, максимизирующие суммарную прибыль при ограничениях на
доступные ресурсы и спрос. Решение строится средствами библиотеки **PuLP**
(решатель CBC).

## Исходная постановка
Мебельная фабрика выпускает два вида столов (офисный и премиум). Заданы прибыль
на единицу, расход ДСП и труда на каждую единицу, запасы ресурсов и
ограничение спроса на премиум-столы. Нужно найти план выпуска с максимальной
прибылью.

## Как обобщено
- Жёстко зашитые две переменные заменены на **произвольное число продуктов и
  ресурсов**; модель строится в цикле по данным.
- Все числовые значения вынесены из кода в JSON (`data/default.json`), который
  скрипт подгружает по умолчанию.
- Добавлена поддержка **бесконечного (неограниченного) спроса**: значение
  `null` в списке `demand` означает отсутствие верхней границы на выпуск.
- Добавлено поле `sense` (`max`/`min`) — задачу можно решать и на минимум.
- Добавлена проверка согласованности размеров (`validate`).

## Специфичные настройки
- `--integer` — выпускать только целочисленные объёмы продукции
  (переменные становятся целыми).
- В данных:
  - `demand[i] = null` — спрос на продукт `i` не ограничен;
  - `usage` — матрица «ресурсы × продукты» (строка — ресурс, столбец — продукт);
  - `currency` — подпись денежных единиц в выводе.

## Файлы данных
- `data/default.json` — исходная задача про мебельную фабрику.
- `data/variation_many_products.json` — четыре продукта и три ресурса:
  демонстрирует масштабирование модели.
- `data/variation_scarce_labor.json` — та же фабрика, но резко урезан ресурс
  труда: показывает, как меняется оптимальный план при дефиците ресурса.

## Запуск
```bash
python task1_production_planning.py
python task1_production_planning.py --input data/variation_many_products.json
python task1_production_planning.py --input data/variation_many_products.json --integer
python task1_production_planning.py --interactive
```
