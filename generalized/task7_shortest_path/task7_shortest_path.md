# Задача 7. Кратчайший путь в графе

## Тип задачи
Задача теории графов: поиск кратчайшего (по сумме весов) пути между двумя
вершинами взвешенного графа. Решается алгоритмом Дейкстры через библиотеку
**NetworkX**.

## Исходная постановка
Задан ориентированный взвешенный граф из 6 вершин. Нужно найти кратчайший путь
из вершины 1 в вершину 6 и его длину.

## Как обобщено
- Граф задаётся **произвольным списком рёбер** в `data/default.json`.
- Добавлен флаг `directed` в данных: граф может быть **ориентированным или
  неориентированным**.
- Вершины могут быть как числами, так и строками (например, названия городов).
- Добавлены проверки: неотрицательность весов (условие применимости Дейкстры),
  наличие начальной и конечной вершин в графе, существование пути между ними.

## Специфичные настройки
- Отдельных флагов командной строки нет — поведение определяется данными.
- В данных:
  - `directed` — `true` (ориентированный) или `false` (неориентированный);
  - `edges` — список рёбер вида `[откуда, куда, вес]`;
  - `source`, `target` — начальная и конечная вершины.

## Файлы данных
- `data/default.json` — исходный ориентированный граф из 6 вершин.
- `data/variation_undirected_cities.json` — неориентированный граф дорог между
  городами с именованными вершинами: демонстрирует строковые вершины и
  неориентированный режим.
- `data/variation_grid.json` — более крупный ориентированный граф из 9 вершин:
  показывает масштабирование.

## Запуск
```bash
python task7_shortest_path.py
python task7_shortest_path.py --input data/variation_undirected_cities.json
python task7_shortest_path.py --input data/variation_grid.json
python task7_shortest_path.py --interactive
```
