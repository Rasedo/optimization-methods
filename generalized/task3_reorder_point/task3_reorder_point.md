# Задача 3. Управление запасами: точка заказа (ROP)

## Тип задачи
Вероятностная модель управления запасами. При случайном спросе нужно
определить **точку заказа** (reorder point) — уровень запаса, при достижении
которого следует размещать новый заказ, чтобы с заданной вероятностью
(уровнем сервиса) не допустить дефицита за время поставки. Используется
квантиль нормального распределения из **scipy.stats**.

## Исходная постановка
Известны средний недельный спрос, его стандартное отклонение, время выполнения
заказа (в неделях) и требуемый уровень сервиса 95%. Нужно рассчитать страховой
запас и точку заказа.

## Как обобщено
- Все параметры вынесены в `data/default.json`.
- Коэффициент надёжности z вычисляется из **произвольного уровня сервиса**
  через `norm.ppf`, а не берётся фиксированным.
- Добавлены настраиваемые единицы измерения товара и времени для вывода.
- Добавлена проверка корректности входных данных (неотрицательность,
  уровень сервиса строго внутри интервала (0, 1)).

## Специфичные настройки
- Отдельных флагов нет — поведение полностью определяется данными.
- В данных:
  - `mean_demand`, `std_demand` — среднее и СКО спроса за период;
  - `lead_time` — время поставки в тех же периодах;
  - `service_level` — требуемый уровень сервиса (доля, например 0.95);
  - `units`, `time_units` — подписи единиц в выводе.

## Файлы данных
- `data/default.json` — исходная задача (уровень сервиса 95%).
- `data/variation_high_service.json` — те же параметры, но уровень сервиса 99%:
  показывает рост страхового запаса при ужесточении требований.
- `data/variation_volatile_long_lead.json` — другой товар с высокой
  волатильностью спроса и длинным сроком поставки: демонстрирует влияние
  разброса спроса и времени поставки на ROP.

## Запуск
```bash
python task3_reorder_point.py
python task3_reorder_point.py --input data/variation_high_service.json
python task3_reorder_point.py --input data/variation_volatile_long_lead.json
python task3_reorder_point.py --interactive
```
