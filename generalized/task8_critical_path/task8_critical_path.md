# Задача 8. Сетевое планирование: метод критического пути (CPM)

## Тип задачи
Сетевое планирование проектов. По списку работ с длительностями и отношениями
предшествования нужно определить минимальную длительность проекта, критический
путь и временные резервы каждой работы. Сетевой график моделируется как
направленный ациклический граф средствами **NetworkX**; расчёт ведётся прямым
и обратным проходом (ES/EF и LS/LF).

## Исходная постановка
Семь работ A–G с заданными длительностями и предшественниками. Нужно найти
длительность проекта, критический путь и резервы времени работ.

## Как обобщено
- Список работ (имя, длительность, предшественники) вынесен в
  `data/default.json`; поддерживается **произвольное число работ** и любая
  структура зависимостей.
- Расчёт ES/EF/LS/LF и резервов выполняется по топологическому порядку, что
  корректно работает для любого DAG.
- Имена работ могут быть содержательными строками (этапы проекта), а не только
  буквами.
- Добавлены проверки: уникальность имён работ, существование указанных
  предшественников, отсутствие циклов в сетевом графике.
- Настраиваемые единицы измерения времени в выводе.

## Специфичные настройки
- Отдельных флагов нет — поведение определяется данными.
- В данных:
  - `tasks` — список объектов `{name, duration, predecessors}`;
  - `time_units` — подпись единиц времени в выводе.

## Файлы данных
- `data/default.json` — исходный проект из 7 работ A–G.
- `data/variation_software_project.json` — план разработки ПО с осмысленными
  названиями этапов и параллельными ветками (бэкенд/фронтенд): демонстрирует
  содержательные имена и ветвление.
- `data/variation_parallel.json` — проект с тремя параллельными линиями разной
  длительности, сходящимися в сборку: наглядно показывает, как определяется
  критический путь среди параллельных работ.

## Запуск
```bash
python task8_critical_path.py
python task8_critical_path.py --input data/variation_software_project.json
python task8_critical_path.py --input data/variation_parallel.json
python task8_critical_path.py --interactive
```
