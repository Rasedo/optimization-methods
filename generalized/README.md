# Обобщённые решения задач по исследованию операций

Эта директория содержит **модифицированные** версии решений восьми
практических задач. Исходные решения (см. директорию `../original`) были
привязаны к конкретным числам из условий. Здесь они обобщены так, чтобы один и
тот же скрипт решал целый класс однотипных задач: размерность, коэффициенты и
прочие параметры задаются данными, а не зашиты в код.

## Структура

```
generalized/
├── README.md                     (этот файл)
├── task1_production_planning/
│   ├── task1_production_planning.py
│   ├── task1_production_planning.md   (описание задачи и способа обобщения)
│   └── data/
│       ├── default.json               (данные по умолчанию)
│       ├── variation_*.json           (две демонстрационные вариации)
│       └── variation_*.json
├── task2_transportation/
│   └── ...
└── ... (аналогично для задач 3–8)
```

В каждой директории задачи лежат: обобщённый скрипт, файл `.md` с описанием
задачи и того, как именно она обобщена (а также специфичных для неё настроек),
и папка `data` с тремя JSON-файлами — дефолтными данными и двумя вариациями,
демонстрирующими универсальность решения.

## Общие флаги

Все восемь скриптов используют единый интерфейс командной строки:

- `--input ПУТЬ` — загрузить данные задачи из указанного JSON-файла вместо
  дефолтного.
- `--interactive` — ввести все данные вручную через диалог в консоли.
- `--dump-default` — напечатать дефолтные данные задачи в формате JSON и выйти
  (удобно как шаблон: перенаправьте вывод в файл, отредактируйте числа и
  подайте обратно через `--input`).

Дополнительно у некоторых задач есть свои флаги (например, `--integer` в
задачах 1 и 2 для целочисленного решения). Они описаны в `.md` соответствующей
задачи.

Запуск без флагов берёт данные из `data/default.json` этой задачи.

## Механизм JSON для подгрузки данных

Дефолтные данные **не зашиты в код** — каждый скрипт при обычном запуске читает
файл `data/default.json` из своей директории. Порядок выбора источника данных:

1. если указан `--input ПУТЬ` — данные берутся из этого файла;
2. иначе если указан `--interactive` — данные вводятся вручную;
3. иначе — читается `data/default.json`.

Чтобы решить свою задачу, проще всего получить шаблон и отредактировать его:

```bash
cd task1_production_planning
python task1_production_planning.py --dump-default > data/my_case.json
# отредактируйте data/my_case.json
python task1_production_planning.py --input data/my_case.json
```

Формат JSON для каждой задачи и смысл всех полей описаны в её `.md`-файле, а
готовые примеры — в файлах `data/variation_*.json`.
